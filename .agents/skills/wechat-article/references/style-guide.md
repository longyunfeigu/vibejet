# 公众号爆款文章风格参考

## 标杆文章分析

### 案例：有道宝库推广文（Karpathy 热点切入）

**写法拆解：**

1. **开头**：蹭 Karpathy 推文热度（1500 万人看了），一句话建立权威感和时效性
2. **过渡**：从"别人在用脚本"到"大多数人需要产品"，制造需求缺口
3. **主体**：不说"产品有 X 功能"，而是"我拿它跑了一个真实调研场景"，全程第一人称实测
4. **功能展示**：每个功能都在使用过程中自然浮现——"先把资料扔进去""然后开始追问""让资料长出新形状"
5. **截图节奏**：每 300-500 字一张图，图上方有引导语，下方有解读
6. **诚实段落**："说实话，那个产品还没出现"——在吹了一通之后主动降温，建立信任
7. **对比**：不是直接贬低竞品，而是"NotebookLM 在国内访问不便……对中文用户来说，这个更接近可用答案"
8. **收尾**：回到 Karpathy 原话做呼应，平衡收场，不过度煽情

**关键句式收集：**

- "接下来发生的事挺有意思。"
- "这里有个体验和 X 的方案根本不同。"
- "说实话我自己用 X 不难，但每次有人问……"
- "不是说它绝对不会 X，而是它把 Y 摆在你面前了。"
- "这件事的价值，不在于它有多完美，在于它真的存在。"

### 通用爆款结构模板

```
Hook（100 字）
  → 热点/数据/个人故事引入

背景铺垫（200 字）
  → 这个问题为什么存在、为什么难、别人怎么解决的

"所以我做了 / 发现了 / 试了这个"（转折点）

实测走一遍（800-1200 字，核心段落）
  → 我做了什么 → 发生了什么 → 为什么有意思
  → 每个功能点在使用叙事中自然浮现
  → 图文交织

诚实评价（200-300 字）
  → 主动说不足
  → 和同类对比
  → 一句话定位

收尾（200 字）
  → 链接 / 如何获取
  → 动机（感性但克制）
  → 互动提问
```

## 禁忌清单

| 禁忌 | 替代 |
|------|------|
| "震惊！""万万没想到！" | 用具体数字或场景制造惊讶 |
| "本产品支持……" | "我发现它能……" |
| "核心竞争力" | "跟 X 不一样的地方在于" |
| "赋能""抓手""闭环" | 说人话 |
| 功能一、功能二、功能三 | 在使用叙事中自然展开 |
| "未来我们计划……" | 如果还没做到就别提 |
| 一段话超过 5 行 | 拆成 2-3 个短段落 |
| 文末才放所有截图 | 图文交织，300-500 字一张 |

## 情绪词库（用于标题和关键句）

### 痛点类
挨打、翻车、哑口无言、被怼、尴尬、踩雷、翻了车、硬仗

### 惊喜类
有意思、没预期到、精确到、看得见、量化、一眼看出

### 行动类
试试、拿去用、先练一轮、提前、彩排、模拟、陪练

### 诚实类
说实话、目前的不足、取决于、做不到、还没出现、不是完美的
