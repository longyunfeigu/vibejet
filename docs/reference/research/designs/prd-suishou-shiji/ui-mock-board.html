API: 反查出的新端点需求——目标读写（如 GET/PUT /api/v1/goals，字段 daily_calories + effective_from）。估算建议值是纯前端公式（如 Mifflin-St Jeor），不需要端点。
决策: R2"改目标不追溯"→ 目标必须带生效日期做版本化（历史日期用当日生效的目标判达成），只存单值会改写历史——这是 Epic 3 最容易做错的数据设计点；MVP 未设置目标时首页用默认值顶替（§11.1）。
⚠️ 推断: 首次引导以整屏表单呈现（非多步 wizard）——单用户工具，两个输入不值得分步。
